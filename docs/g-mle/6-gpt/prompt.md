



문서 스타일에 맞춰 바꿔줘.

  요구사항:
  1) 섹션 제목은 유지한다. (예: ## 1. 준비 조건)
  2) 본문은 여러 단계/목록으로 나누지 말고, **하나의 ```bash``` 코드블록**으로 통일한다.
  3) 블록 안에서 각 단계는 주석으로 번호를 남긴다. 형식: `# 1-1. ...`
  4) 설치 → 실행 → 점검 순으로 정렬한다.
  5) 각 항목은 실제 실행 가능한 명령이어야 하고, 필요 시 대체 명령(예: `X || Y`)도 포함한다.
  6) 끝에 한 줄 요약을 붙인다. (예: “해당 블록은 ... 점검 흐름입니다.”)

  입력으로 줄 섹션:
  - 헤더 + 기존 문구
  - 현재 있는 체크리스트/명령어

  출력은:
  - 기존 내용 삭제 후 동일 제목 아래에 정리된 코드블록 형태로만 치환해줘.
