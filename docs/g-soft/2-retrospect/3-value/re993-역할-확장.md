# 개발자의 사고력 - 역할 확장  

역할에 대한 고민과 확장이 필요하다.  
- 업무 프로세스를 생각해보면 잘 아는 사람이 도맡아서 진행하는 경우가 많다.  
- 잘 아는 사람에게는 프로젝트 완수의 미션이나 기술 리딩에 미션이 주어진다.  
- 결국에 역할에는 여러 책임이 붙는다.  

역할을 받기 위해서는 내가 가진 여러 책임을 확장해야 한다.  
- 개발, 운영, 커뮤니케이션 등 여러 책임들이 존재한다.  


현재 사람들의 역할과 책임을 확인하기  
- 모든 사람들이 어떻게 일을 하고 있는지 큰 그림을 파악해야한다. 조직, 팀 관점이 필요.  
- Output : 역할과 책임, 보고 체계 리스트업
